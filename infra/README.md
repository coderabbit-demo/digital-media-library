# Infrastructure (Terraform) — Digital Media Library

All GCP resources for the `001-auth-activity-feed` feature are provisioned here
with the `hashicorp/google` provider (Terraform >= 1.6). Region defaults to
`us-central1`, sized smallest-by-default for ~100 concurrent users with every
sizing knob parameterized for easy scale-up.

## What gets created

- **Artifact Registry** (Docker) for the API image
- **VPC** + subnet, **Serverless VPC Access connector**, and **Private Services
  Access** (servicenetworking) peering
- **CloudSQL PostgreSQL** (private IP, parameterized tier/HA) + database + user
- **Memorystore for Redis** (Basic tier) — gated by `enable_redis`
- **Cloud Run v2** API service (least-privilege runtime SA, VPC egress, secrets
  from Secret Manager, LB-only ingress)
- **SPA hosting**: Cloud Storage bucket + global external HTTPS Load Balancer +
  Cloud CDN, URL map routing `/api/*` to Cloud Run and everything else to the
  bucket; Google-managed TLS cert (when `domain` is set)
- **Secret Manager**: DB password, session signing key, DATABASE_URL, REDIS_URL
  (generated by Terraform) and the Google OAuth client secret (value added
  out-of-band)
- **Cloud Logging**: dedicated log bucket (retention) + project sink

## Prerequisites

- Terraform >= 1.6
- `gcloud` authenticated with rights to create the above (`gcloud auth
  application-default login`)
- A GCP project with billing enabled (the run/sqladmin/redis/etc. APIs are
  enabled by Terraform)
- A Google OAuth 2.0 client (for `GOOGLE_CLIENT_ID` and the client secret)
- A built API container image pushed to the Artifact Registry repo (see outputs)

## Usage

```bash
terraform init
terraform plan  -var project_id=YOUR_PROJECT_ID
terraform apply -var project_id=YOUR_PROJECT_ID
```

Or copy `terraform.tfvars.example` to `terraform.tfvars`, fill it in, and run
`terraform plan` / `terraform apply`.

### Populate the OAuth client secret (required, out-of-band)

Terraform creates the secret *container* but not its value (the value is issued
by Google and never stored in source). After the first apply:

```bash
printf '%s' 'THE-CLIENT-SECRET-FROM-GOOGLE-CONSOLE' | \
  gcloud secrets versions add dev-google-oauth-client-secret \
    --project=YOUR_PROJECT_ID --data-file=-
```

(Replace `dev` with your `env` value.) Re-deploy/restart Cloud Run if it started
before the secret had a version.

### Build & push the API image

```bash
REPO=$(terraform output -raw artifact_registry_repo)
docker build -t "$REPO/api:v1" ../backend
docker push "$REPO/api:v1"
terraform apply -var project_id=YOUR_PROJECT_ID -var image_tag=v1
```

### DNS / TLS

Set `-var domain=app.example.com` to provision a Google-managed certificate and
an HTTP->HTTPS redirect, then point an A record at the `load_balancer_ip` output.
Managed certs take a few minutes to become ACTIVE after DNS resolves.

## Cost controls

Smallest tiers are the defaults. To minimize standing cost:

- `enable_redis = false` — removes the Memorystore standing charge
- `cloud_run_min_instances = 0` — scale to zero (default; cold starts on idle)
- `cloudsql_tier = "db-f1-micro"` and `cloudsql_ha = false` (defaults)

Scale up by raising `cloud_run_max_instances`, `cloudsql_tier`,
`redis_memory_size_gb`, or enabling `cloudsql_ha` — all variable changes.

## Notes

- `.terraformignore` is committed; state and `*.tfvars` (except `*.tfvars.example`)
  are ignored. Configure a remote backend (e.g. GCS) for team use.
- CloudSQL has `deletion_protection = true` by default; set it false to destroy.
